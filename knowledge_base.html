üìñ User Guide

Step-by-Step Process

Step 1: Change Details

Purpose: Define basic change information and planning

  • Complete change title and description
  • Select change type (Normal/Emergency)
  • Choose requester and assigned agent
  • Define implementation, validation, and backout plans
  • Set planned start/end dates

Step 2: Risk Assessment

Purpose: Evaluate change risk using standardized criteria

Complete assessment across 5 factors (each scored 1-3):

  • Business Impact: Effect on business operations
  • Affected Users: Number of users impacted
  • Complexity: Technical complexity level
  • Testing Level: Extent of testing performed
  • Rollback Capability: Ease of rollback procedures

Step 3: Impacted Assets

Purpose: Associate assets and identify stakeholders

  • Use services dropdown for quick selection of configured asset types
  • Search for additional assets using the general search
  • Review automatically identified technical owners
  • Verify stakeholder completeness

Step 4: Review & Submit

Purpose: Final validation and submission

  • Review comprehensive change summary
  • Verify risk assessment and asset associations
  • Confirm implementation readiness
  • Submit for processing

Risk Scoring Matrix

Criteria Score 1 (Low) Score 2 (Medium) Score 3 (High)

Affected Users < 50 users 50-200 users > 200 users

Testing Level Comprehensive testing Limited testing, some risk Minimal testing, high uncertainty
Rollback Capability Easy rollback, automated Moderate rollback complexity Difficult rollback, potential data loss

Risk Level Determination

LOW RISK Score 5-7: Standard approval process, 2+ days lead time
MEDIUM RISK Score 8-11: Peer review required, 5+ days lead time
HIGH RISK Score 12-15: Peer review + CAB approval, 10+ days lead time

Key Automation Features

  • Automatic SME Assignment: System identifies appropriate Subject Matter Expert for peer review coordination
  • Workflow Triggers: Freshservice workflow automator monitors peer review task completion
  • Status Transitions: Automatic movement from "Pending Review" to "Pending Approval"
  • Approval Creation: Automated generation of approval tickets for technical owners and CAB

‚öôÔ∏è Configuration

Installation Parameters

During app installation, configure the following parameters:

Parameter Description Required Default
freshservice_domain Your Freshservice domain (e.g., company.freshservice.com) Yes -
api_key Freshservice API key with appropriate permissions Yes -
assetTypeNames Comma-separated asset type names for services dropdown No "Software, IT Software, ISP"
plan_type Freshservice plan for API rate limit configuration Yes starter
workspace_id Workspace ID for change management No 2

Asset Type Configuration

To configure asset types for the services dropdown:

  1. Navigate to Admin ‚Üí Asset Management ‚Üí Asset Types in Freshservice
  2. Note the names of asset types you want to include
  3. Enter these names (comma-separated) in the assetTypeNames field during installation
  4. Examples: "Software, IT Software, Database" or "Hardware, Servers, Network Equipment"
